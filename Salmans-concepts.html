Key Concepts for Responsive Web Design

Below are the core terms and ideas that show up in my research on responsive web design. Understanding these concepts helps explain how we built our site to be mobile friendly.

  1. Responsive Web Design (RWD): An approach where a single website automatically adjusts its layout, images, and content to work well on phones, tablets, and desktops.
  2. Viewport Meta Tag: A special HTML tag (for example <meta name="viewport" content="width=device-width, initial-scale=1.0">) that tells the browser how to scale the page on mobile devices.
  3. Fluid Grid Layout: A layout built with relative units (like percentages) instead of fixed pixels, allowing columns to grow or shrink based on the available screen width.
  4. Flexible Images: Images styled so they do not overflow their containers on small screens, usually by applying max-width: 100% and auto height.
  5. Media Queries: CSS rules that only apply under certain conditions, such as a maximum or minimum screen width. Media queries are what let us change layouts for mobile vs. desktop.
  6. Mobile-First Design: A design philosophy where we start by designing for small screens and layer on additional styles for larger breakpoints. This encourages simplicity and performance.
  7. Hamburger Menu: A menu icon (usually three horizontal lines) that toggles navigation open and closed on mobile screens, saving vertical space.
  8. Breakpoints: Specific width ranges where the layout changes. For example, under 768px we might switch to a single column and stack the navigation vertically.
  9. Accessibility in RWD: Making sure responsive sites are still usable with screen readers, keyboard navigation, and proper color contrast. RWD should not break accessibility.
